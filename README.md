# Project Management API

A robust, containerized RESTful API for project and task management. Built with **FastAPI**, **PostgreSQL**, **SQLAlchemy**, and **Docker**, utilizing a clean layered architecture for maintainability and scalability.

## 🌟 Key Features

- **JWT Authentication**: Secure endpoints with JSON Web Tokens and password hashing (bcrypt).
- **Layered Architecture**: Strict separation of concerns (Controllers -> Services -> Repositories).
- **Project Management**: Create, read, update, and delete projects.
- **Task Tracking**: Nested task CRUD operations specific to each project.
- **Data Validation**: Comprehensive request validation using Pydantic schemas.
- **Auto-generated Documentation**: Rich Swagger UI automatically generated by FastAPI.
- **Docker Ready**: Complete `docker-compose` setup with health checks.
- **End-to-End Testing**: Test suite built with `pytest` using a separate SQLite in-memory database.

## 🏗 Architecture

This API follows a layered architecture pattern:

1. **Controllers (`src/controllers/`)**: Handle HTTP requests and responses, route definitions, and OpenAPI metadata.
2. **Services (`src/services/`)**: Contain the core business logic, validation, and authorization checks.
3. **Repositories (`src/repositories/`)**: Handle all database interactions and abstract away SQLAlchemy queries.
4. **Models (`src/models/`)**: SQLAlchemy database schemas and relationships.
5. **Schemas (`src/schemas/`)**: Pydantic models for request/response serialization and validation.

## 🚀 Quickstart (Docker)

The easiest way to run the API and its database is using Docker.

### Prerequisites
- Docker and Docker Compose installed.

### Setup

1. Clone the repository and navigate into it:
   ```bash
   git clone https://github.com/SivaGaneshv1729/project-management-api.git
   cd project-management-api
   ```

2. Start the services using Docker Compose:
   ```bash
   docker-compose up -d --build
   ```

3. The API will now be running at `http://localhost:8000`.
4. **Interactive API Documentation** (Swagger UI) is available at `http://localhost:8000/docs`.

## 💻 Local Setup (Without Docker)

If you prefer to run the API locally using Python and a local database.

### Prerequisites
- Python 3.10+
- PostgreSQL or SQLite (depending on your setup)

### Setup

1. Create a virtual environment and activate it:
   ```bash
   python -m venv venv
   source venv/bin/activate  # On Windows use `venv\Scripts\activate`
   ```

2. Install dependencies:
   ```bash
   pip install -r requirements.txt
   ```

3. Set up your environment variables (create a `.env` file from `.env.example`).
   ```bash
   cp .env.example .env
   ```

4. Run the FastAPI development server:
   ```bash
   uvicorn src.main:app --reload
   ```

## 🧪 Running Tests

The project includes an end-to-end testing suite using `pytest`. The tests are configured in `tests/conftest.py` to use a temporary SQLite database, so they won't interfere with your main database.

```bash
pytest
```

## 📚 API Reference

Here are the primary endpoints available in the API. For a complete and interactive list with request/response schemas, visit `http://localhost:8000/docs`.

### Authentication
- `POST /api/auth/register`: Register a new user account.
- `POST /api/auth/login`: Authenticate and receive a JWT token.
- `GET /api/users/me`: Retrieve current user profile.

### Projects
- `POST /api/projects`: Create a new project.
- `GET /api/projects`: List all projects owned by the user.
- `GET /api/projects/{id}`: Get a specific project.
- `PUT /api/projects/{id}`: Update a project.
- `DELETE /api/projects/{id}`: Delete a project and its tasks.

### Tasks
- `POST /api/projects/{projectId}/tasks`: Create a new task in a project.
- `GET /api/projects/{projectId}/tasks`: List all tasks for a project.
- `GET /api/tasks/{id}`: Get a specific task.
- `PUT /api/tasks/{id}`: Update a task's details or status.
- `DELETE /api/tasks/{id}`: Permanently delete a task.
